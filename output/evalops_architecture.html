Deterministic Eval
exact checks — no LLM needed
String match · regex · schema validation · unit test · CI gate
Exact / Contains Match
· response.contains(expected_fact)
· zero ambiguity — pass or fail
· best for facts, numbers, names
str match
Regex Pattern Check
· re.search(pattern, response)
· flexible: dates, codes, URLs, prices
· no hallucination possible on format
regex
JSON / Schema Validation
· jsonschema.validate(output, schema)
· structured output contracts enforced
· catches missing fields instantly
schema
CI Unit Test Gate
· pytest over agent outputs
· each test = one golden example
· fails the build on any regression
pytest

02
LLM-as-Judge at Scale
probabilistic eval — rubric-based scoring
Correctness · groundedness · coherence · safety · pass@k
Single-criterion Judge
· judge scores one dimension per call
· rubric: 1–5 with explanations
· Phase 4c pattern, now batched
Multi-criterion Rubric
· correctness + groundedness + tone
· JSON: {criterion: score, reason}
· weighted aggregate → final score
pass@k Sampling
· run agent k times on same input
· P(pass) = 1 − (1−p)^k
· reveals variance, not just mean quality
k = 3–10
majority@k / Voting
· majority vote across k outputs
· more robust than single-sample
· useful for open-ended generation tasks
Judge loop:  question ×k → → → agent() ×k → → → judge each → → → pass@k score → → → dashboard
↓
03
Continuous Eval Ops Loop
production failures → regression tests automatically
The flywheel: every failure grows the eval dataset
Production Monitor
· log every agent response in prod
· tag: user_id, session, timestamp
· failure detection: correction / low score
Failure Triage
· classify: wrong fact / hallucination / refusal
· auto-create regression test from failure
· dataset grows with every incident
Versioned Dataset
· curated golden + captured failures
· versioned: v1.0, v1.1, ...
· never shrinks — only grows over time
CI Eval Gate
· eval run triggered on every PR
· score must be ≥ baseline to merge
· blocks regression before it ships
Flywheel:  prod failure → → → triage → → → add to dataset → → → CI eval run → → → score ≥ baseline? → → → ship / block
